# 项目目标 — 民警随手记

## 核心问题

基层民警在日常工作中产生大量笔记、案件记录、巡查信息，但这些信息：
- 散落在各个地方（笔记本、手机、微信群）
- 难以检索和关联
- 无法自动发现串并案线索
- 经验技巧难以积累和传承

## 解决方案

建立一个**智能警务知识库**：
1. 民警像平时一样随手记（语音、文字、图片）
2. LLM 自动将笔记组织成结构化的知识
3. 自动发现人物、地点、案件之间的关联
4. 支持对话式查询和智能检索

## 使用场景

### 日常巡逻
```
民警记录："XX小区今天发现可疑人员，在电动车充电棚附近徘徊"
↓ LLM 自动处理
→ 创建/更新地点页：XX小区
→ 创建/更新人物页：可疑人员（待确认身份）
→ 关联到案件：如果该小区有未破盗窃案
```

### 案件办理
```
民警记录："办理一起盗窃案，嫌疑人在XX小区盗窃电动车一辆"
↓ LLM 自动处理
→ 创建案件页：2024-盗窃-小区电动车
→ 创建人物页：嫌疑人（如有姓名）
→ 关联地点：XX小区
→ 关联法规：治安管理处罚法第49条
→ 检查：该地点是否有其他类似案件？
```

### 串并案分析
```
民警查询："帮我查最近XX小区的所有盗窃案"
↓ LLM 检索知识图谱
→ 返回：3起盗窃案记录
→ 提示：同一地点、相似手法，建议串并案
→ 自动生成研判 summary
```

### 经验传承
```
老民警记录："视频追踪要注意这几个关键点..."
↓ LLM 自动处理
→ 创建技战法页面：视频追踪技战法
→ 关联相关案件
→ 其他民警可通过搜索或对话获取
```

## 工作原则

1. **简单录入** — 民警不需要改变记录习惯
2. **自动组织** — LLM 负责结构化，不需要手动分类
3. **智能关联** — 自动发现知识之间的联系
4. **方便检索** — 对话式查询，找到想要的信息
5. **持续积累** — 知识库越用越丰富，越用越聪明
