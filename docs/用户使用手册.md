## 0. 知枢是什么

> 一句话：**把你的 Word / PDF / 图片 / 笔记，自动整理成可检索、可关联、可对话的知识网络。**

- **本地优先**：你的文档不上云端。云端只管账号 / 邀请码 / 公共配置。
- **LLM 自动整理**：拖文件进去，AI 自动抽人物、案件、地点、证据，建立 `[[wiki 链接]]`。
- **关系图谱 + 对话查询**：可视化看到实体之间的关联；可以用自然语言问"老张和这个案件有没有关系"。
- **多人协作**：每个人有独立的本机库；云端做账号统一管理。

![整体架构示意（占位）](images/01-architecture.png)

## 1. 准备工作

### 1.1 你需要

- 一台 Windows 10 或 11 电脑
- 一个**邀请码**（找管理员要）—— 类似 `K9T3WTNV` 的 8 位字符串
- 一份**大模型 API key**（OpenAI / DeepSeek / 通义千问等，任选其一）

### 1.2 不需要

- ❌ 你**不需要**自己装 Python（我们的安装包里已经带了）
- ❌ 你**不需要**懂命令行
- ❌ 你**不需要**有公网 IP

## 3. 第二步：下载并安装本机客户端

> **重要**：注册成功后，浏览器里的页面**还不能**正常工作——还需要在你自己电脑上启动一个"本机客户端"，它管你的数据。

### 3.1 找下载按钮

注册页 / 登录页的下半部分有一个蓝色卡片：

> **下载本机客户端**
> 解压后双击 setup.bat，按提示输入用户名即可

![下载客户端按钮（占位）](images/05-download-button.png)

点它，浏览器开始下载 `zsnoot-agent-v1.0.0.zip`（约 70 MB）。

### 3.2 解压

下载完毕后，**右键 zip → 解压到当前文件夹**。

建议解压到：
- `C:\zsnoot-agent\` （所有人都行）
- `D:\zsnoot-agent\` （D 盘有空间的）
- ❌ **不要**解压到中文路径里（如 `桌面\我的资料`）
- ❌ **不要**解压到带空格的路径

解压后的文件夹里应该有：

```
zsnoot-agent\
├── setup.bat       ← 首次安装用
├── start.bat       ← 日常启动用
├── stop.bat        ← 停止用
├── README.txt
├── app.py
├── python_embed\   ← 内嵌 Python（不用动）
└── ...
```

![解压后的目录（占位）](images/06-extracted-folder.png)

### 3.3 双击 `setup.bat`

弹出一个黑底窗口，开始向导：

```
═══════════════════════════════════════════════════
  知枢 · 本机 agent 安装向导
═══════════════════════════════════════════════════

模式：bundled（内嵌 Python，无需系统 Python）

[1-2/5] 内嵌 Python 已就绪，跳过装依赖

[3/5] 配置云端连接...
  检测到 preset.ini（打包时预置的服务器配置），云端地址 + 密钥已自动填好
  你只需要填用户名

云端地址 [当前 / 默认: http://81.69.16.235:8090]:  ← 回车
JWT 密钥 [当前 / 默认: 2818e...]:                  ← 回车
你的云端用户名（要与浏览器登录的一致）:              ← 输入你的用户名，回车
```

**三个问题里前两个都已经预填好——直接回车跳过；只在第三个填用户名（与第 2 步注册时一致）。**

![setup 向导（占位）](images/07-setup-wizard.png)

### 3.4 接受桌面快捷方式

向导最后会问：

```
是否在桌面创建 "知枢" 图标? [Y/n]:    ← 回车（接受）
```

回车，安装完成。

![安装完成（占位）](images/08-setup-done.png)

## 5. 第四步：配置大模型 (LLM)

> 知枢的"自动整理材料"能力依赖大模型。**你需要自己提供一个 LLM API key**——这个 key 只在你这台电脑保存，不上云端。

### 5.1 进"系统配置"

左侧栏点 **系统配置**（齿轮图标）。

![系统配置入口（占位）](images/11-config-entry.png)

### 5.2 填模型参数

主配置区有几栏：

#### 5.2.1 入库模型（最重要）

| 字段 | 说明 | 例 |
|------|------|---|
| API key | 你的密钥 | `sk-xxxxxxxxx` |
| Base URL | OpenAI 兼容地址 | `https://api.deepseek.com/v1` |
| 模型 | 模型名 | `deepseek-chat` |
| 提供商 | openai / deepseek 等 | `deepseek` |

![入库模型配置（占位）](images/12-llm-config.png)

#### 5.2.2 视觉模型（可选）

如果要解析图片 / 截图里的文字内容，填一个支持视觉的模型（如 `qwen-vl-max`、`gpt-4o`）。可以和入库模型相同。

#### 5.2.3 其它参数

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 模型温度 | 0.3 | 越低越保守；入库一般 0.2–0.4 |
| max_workers | 3–5 | 并发数；视你 API 限流 |
| Single call 模式 | 开 | 一次 LLM 出多个页面，比 two-call 快 |

### 5.3 测试连接

点右上角 **测试连接** 按钮。

- ✅ **绿色** "测试通过" —— 模型可用
- ❌ **红色** "401 unauthorized" —— key 错了，或 base url 错了
- ❌ **红色** "timeout" —— 网络不通

![测试连接（占位）](images/13-test-llm.png)

测试通过后点 **保存** 按钮。

### 5.4 推荐的 LLM 选择

| 服务商 | 价格 | Base URL | 推荐模型 |
|--------|------|----------|----------|
| DeepSeek（推荐入门）| 便宜 | `https://api.deepseek.com/v1` | `deepseek-chat` |
| 通义千问 | 国内可用 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` |
| OpenAI | 贵但效果好 | `https://api.openai.com/v1` | `gpt-4o-mini` / `gpt-4o` |
| Moonshot (Kimi) | 中等 | `https://api.moonshot.cn/v1` | `moonshot-v1-32k` |

## 6. 开始使用

### 6.1 上传第一份材料

左侧栏 **上传材料**。

![上传材料界面（占位）](images/14-upload-page.png)

把 Word / PDF / 图片 / Markdown / Excel / TXT **拖到上传区**，或点击选择文件。

支持格式：

- 文本类：`.txt`、`.md`、`.html`
- 文档类：`.docx`、`.doc`、`.pptx`、`.pdf`
- 表格类：`.xlsx`、`.csv`
- 图片类：`.png`、`.jpg`、`.bmp`（含 OCR）

### 6.2 等待入库

上传后进入处理流程：

```
1. 解析文件 → 提取纯文本
2. LLM 抽取 → 识别实体（人物 / 案件 / 地点 …）
3. 生成 wiki 页面 → 写入本机 wiki/
4. 建立 [[wiki 链接]] → 双向关联
```

进度条会显示当前在哪一步。**典型一份 5 页 Word 大约 30–90 秒**（取决于 LLM 速度）。

![入库进度（占位）](images/15-ingest-progress.png)

完成后左侧栏 **知识卡片** 下会出现新页面。

### 6.3 看产出的卡片

左栏点开 **知识卡片**，按类目浏览（案件 / 人物 / 地点 / …）。点任意一张卡片：

![卡片视图（占位）](images/16-card-view.png)

每张卡片：
- 上方：YAML frontmatter（结构化字段）
- 中部：正文（可编辑）
- 下方：**反向引用区**——谁引用了这一页

### 6.4 对话查询

左栏点 **对话查询**。输入：

> "老张和黄海街盗窃案有没有关系？"

系统会：
1. 在你的知识库里检索相关页面
2. 把命中页 + 它们的一度邻居塞进 LLM 上下文
3. 生成带 **可点击源引用** 的回答

![对话查询（占位）](images/17-chat.png)

回答自动沉淀为一张 `outputs/` 类型的"问答记忆"页，下次同样问题会优先命中它。

### 6.5 看关系图谱

左栏 **关系图谱**。所有实体被画成一张力导向图。

- **点节点**：高亮它的一度邻居，其余淡化；右侧滑出详情抽屉
- **拖节点**：改位置
- **左上工具栏**：换布局（force / type / community / radial）

![关系图谱（占位）](images/18-graph.png)

## 8. 数据安全

> **承诺**：你的文档 / 笔记 / 知识页**永远不上云端**。

### 8.1 在本机（你这台电脑）

| 位置 | 内容 |
|------|------|
| `~/.handynotes/<你的用户名>/wiki/` | 所有知识页 |
| `~/.handynotes/<你的用户名>/raw/` | 上传的原文归档 |
| `~/.handynotes/<你的用户名>/.env` | **你的 LLM API key** |
| `~/.handynotes/<你的用户名>/data/` | 批次 / 活动日志 |

> Windows 用户可以在文件浏览器输入 `%USERPROFILE%\.handynotes\` 直接打开。

### 8.2 在云端

| 内容 | 你的 |
|------|------|
| 用户名 | ✓ |
| 哈希后的密码 | ✓ |
| 邀请码（一次性）| ✓ |
| 单位 / 职务 | ✓ |
| 选的 schema 模板 | ✓ |
| 心跳计数（页数 / 入库次数）| ✓（数字而已）|
| **你的任何文档 / 笔记 / 卡片** | ✗ **绝不上云**|

### 8.3 LLM 调用

- 用户文档 → **你自己的 API key** → 直连 LLM 服务（点对点；云端不经手）
- 云端 Schema 合成 → **管理员的独立 key**（只发送你输入的"目标 + 对象"几个字，绝不发文档）

## 9. 常见问题排查

### 9.1 浏览器报 "无法连接后端"

> **原因**：本机 agent 没启动 / 已关闭。

**修复**：双击桌面"知枢"图标重新启动。

### 9.2 浏览器报 "Failed to fetch"（个别按钮失效）

> **原因**：浏览器（特别是 Chrome）的 PNA 安全策略拦了。

**修复**：你已经在用 Web V1.0 + 安装包，应该自动解决。如果仍有，按 **Ctrl+Shift+R** 强刷一次浏览器。还不行联系管理员。

### 9.3 报 `agent_not_bound` 或 `wrong_user`

> **原因**：本机 agent 绑定的用户名跟你登录浏览器的不一致。

**修复**：重跑 `setup.bat`，输入正确用户名。

### 9.4 setup.bat 闪一下就关了

> **原因**：解压路径有中文或空格；或杀毒软件拦了。

**修复**：换到 `C:\zsnoot-agent\` 或 `D:\zsnoot-agent\`，临时关杀毒重跑。

### 9.5 LLM "测试连接" 报红

> **原因**：API key 错 / Base URL 错 / 网络问题。

**修复**：
- key 复制时少了字符 → 重粘
- Base URL 末尾**不要带斜杠**：`https://api.deepseek.com/v1` ✓，`https://api.deepseek.com/v1/` ✗
- 公司网络挡了海外 API → 用 DeepSeek / 通义这类国内服务

### 9.6 上传后"待精炼"页面很多

> **原因**：LLM 响应不规范（小模型常见）。

**修复**：
- 换更强的模型试试（gpt-4o-mini 比 deepseek-chat 稍贵但更稳）
- 「知识库体检」→ 占位页 tab → 智能补全

### 9.7 我的数据丢了？

> **预期**：所有数据在 `~/.handynotes/<用户名>/` 文件夹。

**找回**：Windows 文件浏览器输 `%USERPROFILE%\.handynotes\` 看。把这个文件夹定期备份到网盘 / U 盘即可。

### 9.8 换电脑了，数据怎么迁移？

1. 旧电脑：拷贝整个 `C:\Users\<你>\.handynotes\<你的用户名>\` 文件夹
2. 新电脑：装安装包、绑定同样的用户名
3. 把那个文件夹拷到新电脑相同位置（覆盖）
