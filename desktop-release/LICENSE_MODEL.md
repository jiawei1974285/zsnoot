# 商业授权模式和许可证管理设计

## 授权目标

桌面版要同时满足两件事：

- 基层用户可以在无公网、弱网络环境下使用。
- 商业版本可以控制试用期、到期时间、机器数、席位数和高级功能。

因此 A 版采用“离线签名许可证”为主，“在线激活”为后续增强。

## 商业版本建议

| 版本 | 面向客户 | 授权方式 | 典型限制 |
|---|---|---|---|
| 单机专业版 | 个人警务工作者、小团队试用 | 绑定 1 台机器 | 到期时间、功能开关 |
| 单位本地版 | 派出所、分局、专项组 | 绑定单位和席位数 | 机器数、席位数、到期时间 |
| 私有部署版 | 市局、省厅、专案平台 | 项目合同授权 | 私有化部署、接口集成、定制服务 |

## 收费建议

- 试用版：7 到 30 天，可限制导入批次数、知识页数量或 LLM 调用次数。
- 单机专业版：按年授权，适合快速成交。
- 单位本地版：按席位或按机器授权，含基础售后。
- 私有部署版：项目制报价，包含部署、培训、数据迁移和二次开发。

## 许可证文件结构

许可证是一个 JSON 文件，例如 `license/license.json`。核心字段：

```json
{
  "license_id": "lic_demo_001",
  "customer": {
    "name": "Demo Police Unit",
    "contact": "demo@example.com"
  },
  "tier": "organization_local",
  "issued_at": "2026-05-01T00:00:00Z",
  "expires_at": "2027-05-01T00:00:00Z",
  "max_seats": 20,
  "machine_binding": {
    "mode": "fingerprint",
    "fingerprints": ["demo-machine-fingerprint"]
  },
  "features": {
    "ingest": true,
    "graph": true,
    "chat": true,
    "batch_reparse": true,
    "license_admin": true
  },
  "signature_alg": "Ed25519",
  "signature": "base64-signature-placeholder"
}
```

签名覆盖除 `signature` 之外的规范化 JSON 内容。应用内只保存公钥，私钥只保存在授权签发工具或授权服务器中。

## 机器指纹

A 版建议机器指纹只使用稳定且低隐私风险的组合：

- Windows MachineGuid 的哈希值。
- 系统盘序列号的哈希值。
- 主机名的哈希值。

不要明文上传或保存原始硬件信息。许可证中保存的是哈希后的 fingerprint。

## 后端校验点

许可证校验必须在后端执行：

- 应用启动时读取 `desktop-release/license/license.json`。
- 校验 JSON 格式、签名、到期时间、机器指纹和功能开关。
- `/api/auth/status` 返回 license 状态，供前端显示。
- 上传、深度解析、批量重跑、图谱构建、对话问答等商业功能在 API 层检查 license。

前端只能展示授权状态，不能作为授权的安全边界。

## 状态模型

| 状态 | 含义 | 产品行为 |
|---|---|---|
| `valid` | 授权有效 | 正常使用 |
| `trial` | 试用期有效 | 显示剩余天数 |
| `missing` | 未找到许可证 | 允许初始化，限制商业功能 |
| `expired` | 已过期 | 只读模式，提示续费 |
| `machine_mismatch` | 当前机器不在授权范围 | 阻止商业功能 |
| `tampered` | 签名失败或文件被改 | 阻止商业功能并记录日志 |

## 离线签发流程

1. 客户打开授权页，复制机器码。
2. 销售或管理员在内部签发工具输入客户、版本、到期日、机器码和功能。
3. 签发工具生成 `license.json`。
4. 客户把文件放入 `desktop-release/license/license.json`，或通过管理员 UI 上传。
5. 后端重新读取许可证并显示授权状态。

## 在线激活升级路径

后续可增加授权服务器：

- 客户输入激活码。
- 本地应用提交激活码和机器指纹哈希。
- 授权服务器返回签名许可证。
- 本地仍按离线许可证校验运行，断网后也可继续使用。

## 风险和防护

- 不要把签名私钥放进桌面包。
- 不要让前端决定功能是否可用。
- 不要只靠混淆或本地明文配置做商业授权。
- 许可证失败时不要删除用户数据，应进入只读或受限模式。
- 对专案资料场景，授权服务器不能要求上传业务数据。
